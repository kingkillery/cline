## Codebase Patterns
- Kanban/autonomous runtime changes should build on the current Cline-derived execution path rather than introducing a parallel scheduler.
- `src/core/task/tools/handlers/SubagentToolHandler.ts` currently fans out subagent prompts concurrently and is the first runtime-stability seam to harden.
- `cli/src/agent/ClineAgent.ts` already enforces one active prompt per session, which is a useful lower-level isolation primitive but not a full kanban control plane.
- `src/integrations/terminal/standalone/StandaloneTerminalManager.ts` and `src/integrations/terminal/standalone/StandaloneTerminalProcess.ts` already own background command tracking and process termination hooks that should be reused, not replaced.
- Small persisted JSON state is acceptable for MVP orchestration recovery; avoid adding a database.
- This repo may be dirty during task work; unrelated changes must be preserved.
